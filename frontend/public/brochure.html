01 · Platform Overview

One platform. Ten modules. One fiscal backbone.

Quot PSE is a multi-tenant, schema-isolated IFMIS (Integrated Financial Management Information System) purpose-built for Nigerian public-sector finance. Every module reads and writes through a single General Ledger, coded against the Federal National Chart of Accounts — so data flows end-to-end from Appropriation Act to IPSAS financial statements without reconciliation gymnastics.

⚖

Regulation-first

Appropriation Act enforcement, TSA sub-account discipline, FAAC inflow tagging, statutory XML exports and IPSAS-accrual financial statements are core — not modules you configure after the fact.

🧩

Modular & independent

Switch on Budget + GL today; add Procurement or HR when you're ready. Every module stands alone and shares the same ledger, audit trail, approval matrix and NCoA coding.

🛡

Audit-ready by default

Segregation of Duties enforced at the database layer, dual-control overrides require two AG signatures, and every state change is append-only logged with full user and tenant context.

Security & reliability

  • Multi-tenant: schema-per-tenant PostgreSQL — complete data isolation
  • SOD: initiator ≠ approver enforced at DB level
  • Dual control: two AG signatures on every override
  • Append-only audit log: every state change captured
  • Immutable postings: reverse-and-repost pattern; no silent edits
  • Rate limiting: scoped throttles on login, writes, bulk import
  • JSON structured logs + Sentry error capture
  • Prometheus metrics: latency, error rate, queue depth
  • Redis-backed cache: 10-min TTL for hot reports, fail-open
  • Encrypted backups: nightly pg_dump per tenant, rotation + monthly snapshots
  • DR drill: quarterly restore into staging (9-step runbook)
  • Openapi 3.1 schema: Swagger + ReDoc for integrators

Data residency

Tenant data is resident in Nigeria. The Quot PSE platform runs on infrastructure with local availability zones; backups never leave the jurisdiction. Each tenant gets a dedicated PostgreSQL schema, and cross-tenant data access is impossible at the database layer.
